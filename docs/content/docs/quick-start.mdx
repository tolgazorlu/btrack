# Quick Start

This guide walks you through a typical btrack workflow from starting a session to stopping it.

If you'd rather drive everything from a single prompt, run `btrack` on its own to open the [interactive console](/docs/console) — a Gemini-style REPL with `@`-quick actions and free-text AI chat.

## 1. Start a Session

```bash
btrack s "fix login bug"
```

Starts a new tracking session with a description. Alias: `btrack start`.

## 2. Add a Note

```bash
btrack n "found the issue — JWT clock skew between services"
```

Attaches a timestamped note to the currently running session. Alias: `btrack note`.

You can also add notes to past sessions using the `--session` flag:

```bash
btrack n "follow-up: add monitoring" --session 42
```

## 3. Check Live Status

```bash
btrack w
```

Shows a live-updating status display with elapsed time, session description, and any notes. Press `q` to exit.

## 4. Stop the Session

```bash
btrack x                                            # AI offers a message; accept or skip
btrack x -m "fixed JWT clock skew, deployed to staging"
btrack x --no-ai                                    # save without a message
```

Stops the running session. The closing message is optional — running `btrack x` on its own asks AI for a one-line summary; declining saves the session without a message. Alias: `btrack stop`.

## 5. See What Shipped

```bash
btrack shipped
```

Cross-references your last session's time window with `git log` and shows the commits that landed. Pair it with `btrack stop` to do an honest end-of-task self-check. Alias: `btrack sh`.

## 6. View History

```bash
btrack h
```

Shows today's sessions with durations and notes.

```bash
btrack h -w   # this week
btrack h -m   # this month
```
